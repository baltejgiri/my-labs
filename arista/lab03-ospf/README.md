## Background — OSPF Key Concepts
### Router Types
| Type | Description | ------|-------------| Internal Router | All interfaces in the same area | ABR (Area Border Router) | Interfaces in multiple areas — connects areas to backbone 
| |
| ASBR (AS Boundary Router) | Redistributes routes from external routing protocols into OSPF | Backbone Router | Has at least one interface in Area 0 |
### LSA Types (key ones to know)
| LSA Type | Name | Generated By | Flooded To | ----------|------|--------------|------------| Type 1 | Router LSA | Every router | Within the area | Type 2 | Network LSA | DR on 
| broadcast segments | Within the area | Type 3 | Summary LSA | ABR | Between areas | Type 4 | ASBR Summary LSA | ABR | Other areas | Type 5 | AS External LSA | ASBR | Entire OSPF 
| domain | Type 7 | NSSA External LSA | ASBR in NSSA | Within NSSA area only |
### OSPF Area Types
| Area Type | Accepts LSA Types | Use Case | -----------|------------------|----------| Backbone (Area 0) | 1, 2, 3, 4, 5 | Core — all areas must connect here | Standard Area | 1, 
| 2, 3, 4, 5 | Regular area with full routing info | Stub Area | 1, 2, 3 | Blocks external LSAs, uses default route | Totally Stubby Area | 1, 2 | Blocks external + inter-area LSAs 
| |
| NSSA | 1, 2, 3, 7 | Allows redistribution without Type 5 LSAs |
### OSPF Neighbour States (in order)
``` Down → Init → 2-Way → ExStart → Exchange → Loading → Full ``` - **2-Way** — DR/BDR election happens here - **Full** — adjacency is fully established, LSDB is synchronised
### DR/BDR Election Rules
1. Highest OSPF priority wins (default 1, range 0–255; 0 = never DR) 2. Tie-break: highest Router ID wins 3. Router ID is: manually configured > highest loopback IP > highest 
physical IP ---

| Command | What it shows | ---|---| `show ip ospf neighbor` | Neighbour table with state (FULL/2-WAY/INIT etc.) | `show ip ospf neighbor detail` | Full neighbour detail including 
| dead timer countdown | `show ip ospf` | OSPF process info, Router ID, area count, SPF stats | `show ip ospf interface` | Per-interface OSPF config, timers, DR/BDR, area | `show ip 
| ospf interface Ethernet2` | Specific interface OSPF detail | `show ip ospf database` | Full LSDB — all LSA types | `show ip ospf database router` | Type 1 Router LSAs only | `show 
| ip ospf database network` | Type 2 Network LSAs only | `show ip ospf database summary` | Type 3 Summary LSAs only | `show ip route ospf` | OSPF routes in routing table (`O` = 
| intra, `O IA` = inter-area) | `show ip route` | Full routing table | `clear ip ospf process` | Reset OSPF process — forces re-election and re-adjacency |

## Key Takeaways
- OSPF Router ID must be unique across the entire OSPF domain — duplicate IDs cause database inconsistency - All non-backbone areas must connect to Area 0 — this is a fundamental 
OSPF design rule - Hello and dead timers must match between neighbours — mismatch prevents adjacency from forming - DR/BDR election is only relevant on broadcast (multi-access) 
segments — point-to-point links skip it - ABR generates Type 3 Summary LSAs to advertise inter-area routes — this is how routes cross area boundaries - Stub areas block Type 5 
External LSAs; totally stubby areas also block Type 3 Summary LSAs - `O` routes are intra-area; `O IA` routes are inter-area (came through an ABR) - Arista EOS requires `service 
routing protocols model multi-agent` before OSPF will function correctly ---
